Deployment in Options

This section explain how you deploy in QlikView and QlikSense.

Deloying in Qlikview

QlikView Server is a platform for hosting and sharing QlikView information over an intranet or the Internet. QlikView Server connects users, client types, documents, and objects within a secure environment.

QlikView Publisher manages content, access, and distribution. By reducing data, each user can be presented with tailored information. The QlikView Publisher service and user interface are fully integrated into QlikView Server and QlikView Management Console (QMC).

How Do You Deploy QlikView On Application Server?

If the server is Installed on your Laptop

Click On Start > All Programs > QlikView > QlikView Enterprise Management Console

A browser will open.

Go to System Tab > SetUp > QlikView Server > Click on + > Click on QVS @ your pc name >       Folder

Specify the folder where you have you QlikView application.

Click on Apply button at the right bottom. (Donâ€™t forget to click on this button)

This way you can deploy your application on QlikView Server.

 Deploying In Qlik Sense

The Qlik Sense architecture is based on the concept of sites. A Qlik Sense site is a collection of one or more nodes (that is, server machines) connected to a common logical repository or central node. Qlik Sense can be deployed in many ways. This section describes different deployment scenarios.

Deploying Single Nodes Sites

In this deployment scenario, all Qlik Sense services run on a single node. This kind of deployment works best in a single time zone, where reloads of data can be done during the night.

 

Deploying Multi-nodes Sites

In a multi-node site, the site is spread out across two or more nodes that share the same set of data and license key.

The basic steps for deploying a multi-node site are as follows:

Plan the nodes that are needed in the deployment.

Install the first node in the site. This node becomes the central node, which contains all apps that are needed.

Install an additional node as a non-central node.

In the Qlik Management Console (QMC) on the central node, add the new node to the site.

Wait for the first synchronization to complete and then test the new node.

Continue to install additional nodes â€“ one at a time â€“ as described in step 3 until the multi-node site is complete.
